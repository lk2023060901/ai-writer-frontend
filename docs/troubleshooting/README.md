- 使用浏览器开发者工具检查元素位置
- 添加明显的背景色和边框来可视化问题

- 使用Redux DevTools监控状态变化
- 添加action和reducer的调试日志
- 检查组件是否正确订阅状态更新

## 🚨 常见错误模式

### 组件重复渲染
**症状：** DOM中出现多个相同的组件实例
**原因：** 条件渲染逻辑不够严格，多个条件同时为true
**解决方案：** 使用互斥的条件判断，确保组件实例唯一性

### 布局遮挡问题
**症状：** 元素被其他元素部分或完全遮挡
**原因：** 高度计算没有考虑固定定位的元素（如导航栏）
**解决方案：** 使用calc()函数正确计算高度，减去固定元素的高度

### 样式冲突
**症状：** CSS样式没有按预期生效
**原因：** 选择器优先级问题或样式被覆盖
**解决方案：** 使用更具体的选择器或适当的!important声明

## 📋 调试清单

在排查问题时，请按以下顺序进行：

1. **确认问题复现步骤**
   - [ ] 记录详细的操作步骤
   - [ ] 确认问题是否稳定复现
   - [ ] 检查是否在不同环境下都存在

2. **检查控制台错误**
   - [ ] 查看Browser Console是否有JavaScript错误
   - [ ] 检查Network面板是否有请求失败
   - [ ] 确认是否有React组件报错

3. **分析DOM结构**
   - [ ] 检查元素是否正确渲染到DOM中
   - [ ] 验证元素的数量是否符合预期
   - [ ] 确认元素的位置和样式

4. **检查状态管理**
   - [ ] 确认Redux状态是否正确更新
   - [ ] 检查组件是否正确响应状态变化
   - [ ] 验证action是否正确派发

5. **添加调试信息**
   - [ ] 在关键位置添加console.log
   - [ ] 使用明显的视觉标记（如红色背景）
   - [ ] 记录调试过程和发现

6. **清理和验证**
   - [ ] 移除所有调试代码
   - [ ] 验证修复方案的有效性
   - [ ] 进行回归测试
