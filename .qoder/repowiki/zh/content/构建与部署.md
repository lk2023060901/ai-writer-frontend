## 构建与部署概述

本项目采用 Vite 作为前端构建工具，结合 TypeScript 和 React 技术栈，实现了高效的开发与生产构建流程。通过 `vite.config.ts` 配置构建输出、资源优化和代码分割策略；通过 `package.json` 中的 scripts 定义开发、构建和预览命令；通过 `tsconfig.app.json` 提供独立的 TypeScript 编译选项。整体流程支持快速本地开发、类型检查、资源压缩与模块化输出，适用于部署到 Vercel、Netlify 等现代云平台。

## Vite 构建配置详解

`vite.config.ts` 文件定义了项目的核心构建行为，包括路径别名、开发服务器配置、构建输出设置、CSS 源码映射以及环境变量注入。

### 输出路径配置

构建产物的输出目录由 `build.outDir` 指定，当前配置为 `"dist"`，即所有打包文件将生成在项目根目录下的 `dist` 文件夹中。

### 资源压缩与源码映射

- `sourcemap: true` 启用了构建时生成源码映射文件（source map），便于生产环境下的调试。
- Vite 默认使用 Rollup 进行生产构建，自动对 JavaScript、CSS 和静态资源进行压缩优化。

### 代码分割策略

通过 `rollupOptions.output.manualChunks` 配置实现了手动代码分割，将第三方依赖按功能拆分为独立的 chunk：

- `vendor`：包含 React 核心库（`react`, `react-dom`）
- `antd`：Ant Design 组件库
- `redux`：Redux 相关状态管理库（`@reduxjs/toolkit`, `react-redux`）

该策略有助于提升缓存效率，减少主包体积，实现按需加载。

### 环境变量注入

通过 `define` 配置项，将 `__DEV__` 变量注入到代码中，其值为 `process.env.NODE_ENV === 'development'` 的字符串化结果。此变量可在运行时用于条件判断是否处于开发环境。

## Package.json 脚本行为分析

`package.json` 中的 `scripts` 字段定义了项目的标准执行流程，涵盖开发、构建、预览和 lint 检查。

### dev 脚本：本地开发模式

```json
"dev": "vite"
```

启动 Vite 开发服务器，支持热模块替换（HMR），监听文件变化并实时刷新浏览器。开发服务器运行在 `localhost:5173`，配置中 `server.open: true` 表示自动打开浏览器。

适用场景：日常开发调试，快速迭代 UI 与逻辑。

### build 脚本：生产环境构建

```json
"build": "tsc -b && vite build"
```

该脚本执行两个步骤：
1. `tsc -b`：基于 `tsconfig.json` 中的引用关系，先进行 TypeScript 类型检查与编译预处理。
2. `vite build`：调用 Vite 执行实际的打包操作，生成优化后的静态资源。

适用场景：准备部署前的构建，确保类型安全与资源优化。

### preview 脚本：本地预览生产构建

```json
"preview": "vite preview"
```

启动一个本地服务器，用于预览 `dist` 目录中的生产构建产物。此命令不会重新构建项目，仅用于验证构建结果是否正常。

适用场景：部署前最终确认页面功能与样式。

## TypeScript 编译配置

`tsconfig.app.json` 为应用代码提供了独立的 TypeScript 编译选项，与 `tsconfig.node.json` 分离，确保前后端类型配置互不干扰。

### 编译目标与模块系统

- `target: "ES2022"`：编译目标为 ES2022，支持现代 JavaScript 特性。
- `module: "ESNext"`：使用 ES 模块语法，与 Vite 的原生 ES 模块支持保持一致。
- `jsx: "react-jsx"`：启用 React 的新 JSX 转换（automatic runtime），无需手动导入 `React`。

### 路径映射

通过 `paths` 配置启用了 `@/*` 别名，指向 `src/*` 目录，提升导入语句的可读性与维护性。

### 严格类型检查

启用多项严格检查选项：
- `strict`: 启用所有严格类型检查模式
- `noUnusedLocals`: 禁止未使用的局部变量
- `noFallthroughCasesInSwitch`: 防止 switch 语句中 case 落空

这些设置有助于提升代码质量与类型安全性。

### 不生成输出文件

`"noEmit": true` 表示此配置不生成 `.js` 文件，仅用于类型检查。实际构建由 Vite 处理。

## 生产环境构建最佳实践

### CDN 资源优化

建议将静态资源（如 JS、CSS、图片）托管至 CDN，提升全球访问速度。可通过配置 `base` 字段指定资源基础路径：

```ts
export default defineConfig({
  base: 'https://cdn.example.com/ai-writer-frontend/'
})
```

### Gzip 压缩

部署时应启用服务器端 Gzip 压缩，进一步减小传输体积。大多数云平台（如 Vercel、Netlify）默认支持。

### 缓存策略设置

- 对 `dist` 中的静态资源文件（含 hash）设置长期缓存（如 `Cache-Control: max-age=31536000`）
- 对 `index.html` 设置短缓存或不缓存（`Cache-Control: no-cache`），确保用户获取最新版本

可通过 `.vercel` 或 `netlify.toml` 配置文件定义缓存规则。

## 部署至静态服务器或云平台

### 部署流程

1. 执行 `npm run build` 生成 `dist` 目录
2. 将 `dist` 目录内容上传至目标平台
3. 配置域名与环境变量

### Vercel 部署示例

创建 `vercel.json` 配置文件：

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "routes": [
    { "src": "/.*", "dest": "/index.html" }
  ]
}
```

### Netlify 部署示例

在 `netlify.toml` 中配置：

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### 环境变量配置

在云平台控制台中设置环境变量（如 API 地址），Vite 会自动注入以 `VITE_` 开头的变量。例如：

```ts
const apiURL = import.meta.env.VITE_API_URL;
```

### 域名绑定

在平台控制台添加自定义域名，并配置 DNS 解析，即可完成域名绑定。

## 总结

本项目通过 Vite 实现了高效、现代化的前端构建流程。`vite.config.ts` 提供了灵活的构建配置，`package.json` 脚本覆盖了开发全生命周期，`tsconfig.app.json` 确保了类型安全。结合 CDN、Gzip 和缓存策略，可进一步优化生产性能。部署至 Vercel 或 Netlify 等平台简单快捷，支持自动化构建与域名绑定，适合持续集成与交付。
